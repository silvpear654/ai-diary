# ai-diary
팀원 A — 암호화 & 보안 엔진 (The Shield)
팀원 A는 프로젝트의 보안 기반 전체를 책임진다. 사용자가 처음 설정한 비밀번호를 그대로 저장하는 게 아니라, hashlib 라이브러리의 SHA-256 같은 해시 함수를 사용해 복원 불가능한 형태로 변환하고, 이를 로그인 인증에 활용하는 로직을 구현한다. 일기 본문 암호화는 Python의 cryptography 라이브러리를 활용한 AES 방식이나, 학습 목적에서 XOR 기반의 직접 구현 방식 중 하나를 선택해 진행한다. 핵심은 암호화된 파일을 외부에서 직접 열었을 때 절대 내용을 알아볼 수 없어야 한다는 것이다. 복호화 함수도 함께 구현해서, 올바른 키가 있을 때만 원문을 복원할 수 있도록 설계한다.

팀원 B — 파일 시스템 & 데이터 관리 (The Archivist)
팀원 B는 암호화된 일기 데이터를 어떤 구조로 저장하고 불러올지 설계하는 역할이다. Python의 os, pathlib 모듈을 사용해 날짜별 디렉토리 구조를 자동으로 생성하고, 각 일기를 JSON이나 바이너리 포맷으로 저장하는 방식을 구현한다. 또한 키워드나 날짜를 기준으로 일기를 빠르게 검색할 수 있도록 인덱스 파일(index.json 등)을 별도로 관리하는 로직도 담당한다. 다른 팀원들이 데이터를 읽고 쓸 때 사용할 공통 함수(API)를 만들어두는 것도 이 팀원의 몫이다.

팀원 C — AI 감정 분석 & 요약 에이전트 (The Counselor)
팀원 C는 프로젝트에서 가장 "AI다운" 기능을 담당한다. requests 라이브러리나 openai Python SDK를 활용해 Claude나 GPT 같은 외부 AI API에 일기 본문을 전송하고, 감정 분석 결과와 한 줄 요약을 응답으로 받아오는 로직을 구현한다. 단순히 API를 호출하는 것에서 그치지 않고, "이 일기에서 사용자의 주된 감정을 추출하고, 따뜻한 말투로 위로 한마디를 건네줘" 같은 효과적인 프롬프트를 직접 설계하는 것도 핵심 업무다. API 응답이 실패했을 때의 예외 처리도 함께 구현해야 한다.

팀원 D — 명령어 인터페이스 & 파서 (The Gatekeeper)
팀원 D는 사용자가 터미널에 입력하는 모든 명령어를 해석하는 엔진을 만든다. Python의 argparse 모듈을 활용해 vault write, vault read --date 2026-04-30, vault search --tag 연대 같은 명령어를 구조적으로 파싱하고, 각 명령에 맞는 함수를 호출하는 메인 진입점 역할을 한다. 잘못된 명령어나 누락된 인자가 입력됐을 때 사용자에게 친절한 안내 메시지를 출력하는 예외 처리도 이 팀원이 전담한다. 사실상 다른 팀원들이 만든 모듈들을 하나로 연결하는 접착제 역할이기도 하다.

팀원 E — 통계 시각화 & 환경 설정 (The Analyst)
팀원 E는 데이터를 눈에 보이는 형태로 표현하고, 전체 앱의 설정을 관리하는 역할이다. matplotlib 같은 라이브러리나, 순수 터미널 출력만을 사용하는 ASCII 차트 방식으로 최근 일주일간의 감정 변화 추이를 그래프로 시각화하는 기능을 구현한다. 또한 사용자별 테마 색상, 언어 설정, 비밀번호 변경 이력 같은 환경 설정값을 config.json 파일로 저장하고 불러오는 모듈도 만든다. 앱이 처음 실행될 때 설정 파일이 없으면 기본값으로 자동 생성되도록 하는 초기화 로직도 포함된다.
